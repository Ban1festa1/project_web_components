# Пользовательский веб-компонент «Список задач»

## Описание работы

В рамках данной практической работы был разработан пользовательский веб-компонент для управления списком задач. Компонент предназначен для добавления, отображения, редактирования состояния и удаления задач.

Основное внимание в работе уделено оптимизированному обновлению интерфейса. При изменении данных не происходит полная перерисовка списка задач, обновляются только необходимые элементы. Это позволяет повысить производительность и соответствует современным подходам к разработке веб-интерфейсов.

Компонент реализован в виде пользовательского HTML-тега `<task-list>`, что обеспечивает удобство повторного использования на других веб-страницах.

## Цель работы

Целью практической работы является создание пользовательского веб-компонента для отображения и управления списком задач с использованием HTML, CSS и JavaScript, а также реализация оптимизированного механизма рендеринга и обновления данных.

## Задачи работы

В процессе выполнения практической работы были решены следующие задачи:

- разработан пользовательский веб-компонент с использованием технологии Custom Elements;
- реализована форма для добавления новых задач;
- реализовано отображение задач в виде списка;
- реализована возможность отмечать задачи как выполненные;
- реализована функция удаления задач;
- обеспечено оптимизированное обновление интерфейса без полной перерисовки списка;
- проведено тестирование и отладка компонента.

## Используемые технологии

В ходе выполнения работы использовались следующие технологии:

- HTML — для структуры страницы и использования пользовательского тега;
- CSS — для стилизации пользовательского интерфейса и анимаций;
- JavaScript — для реализации логики работы компонента.

Сторонние библиотеки и фреймворки в работе не использовались.

## Структура проекта

index.html — основной HTML-файл;
CSS/styles.css — файл стилей;
JS/main.js — реализация пользовательского веб-компонента.

## Функциональность компонента

Веб-компонент предоставляет следующие возможности:

- добавление новых задач через форму;
- отображение списка задач;
- изменение статуса задачи (выполнена / не выполнена);
- удаление отдельных задач;
- отображение состояния пустого списка;
- подсчёт общего количества задач и количества выполненных.

## Оптимизация рендеринга

Оптимизация обновления интерфейса реализована следующим образом:

- при добавлении задачи в DOM добавляется только новый элемент списка;
- при удалении задачи удаляется только соответствующий элемент;
- при изменении статуса задачи изменяется только её CSS-класс;
- используется делегирование событий, что уменьшает количество обработчиков.

Таким образом, достигается минимизация перерисовок и повышение производительности компонента.

## Тестирование и отладка

В ходе выполнения работы было проведено ручное тестирование:

- проверена работа добавления задач;
- проверена корректность удаления задач;
- проверена работа смены статуса задач;
- проверена корректная работа компонента при пустом списке и при большом количестве задач.

Ошибок в процессе тестирования выявлено не было.
